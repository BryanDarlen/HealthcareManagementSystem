An end-to-end machine learning project for predicting whether a patient will miss a scheduled medical appointment. The project includes data cleaning, feature engineering, model training with LightGBM, an optional FastAPI prediction API, and a Streamlit dashboard for interactive exploration.

## Project Overview

Missed appointments create operational inefficiency and financial loss in healthcare systems. This project predicts appointment no-show risk using patient, scheduling, and historical behavior features so clinics can prioritize reminders and interventions for higher-risk cases.

## What This Project Includes

- Raw appointment dataset
- Data cleaning and preprocessing pipeline
- Feature engineering pipeline
- LightGBM no-show prediction model
- Streamlit dashboard for interactive analysis
- FastAPI app for prediction serving
- Saved model artifacts for reproducibility and demo use

## Dataset Notes

- `data/raw/medical_appointment.csv` is the official raw dataset.
- `data/processed/cleaned_data.csv` is generated by running `python src/data/data_loader.py`.
- `data/features/engineered_features.csv` is a generated, model-ready dataset kept in the repository for convenience and reproducibility.
- `notebooks/01_EDA.ipynb` is exploratory only and is not the official training pipeline.

## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/yourusername/healthcare-no-show-prediction.git
cd healthcare-no-show-prediction
```

### 2. Create a virtual environment and install dependencies

```bash
python -m venv venv
source venv/bin/activate
```

On Windows:

```bash
venv\Scripts\activate
```

Then install packages:

```bash
pip install -r requirements.txt
```

### 3. Run the pipeline

```bash
python src/data/data_loader.py
python src/features/feature_engineer.py
python src/models/train.py
```

### 4. Launch the dashboard

```bash
streamlit run app/dashboard.py
```

## Optional: Run the API

```bash
uvicorn src.api.main:app --reload
```

Then open the docs locally at `http://127.0.0.1:8000/docs`.

## Optional: Run tests

```bash
pytest
```

## Model

The main model used in this project is LightGBM for binary classification of appointment no-show risk.

Saved artifacts:
